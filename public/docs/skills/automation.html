How this proves entry-level system administration

Each evidence entry links to artifacts showing concrete work you performed. For entry-level verification we provide:

  • Commit/patch — shows code you authored (see the attached patch).
  • Execution log — timestamped run output that proves you ran the script and produced the result (`automation-run-2024-10-05.log`).
  • Reproduction steps — exact commands in the evidence YAML so a reviewer can reproduce results locally.
  • Verification — a verifier field and status in the YAML (`Verified`/`Internal`/`Redacted`).

Example quick reproduction (local):

pwsh -NoProfile -Command "./tools/Generate-BaselineReport.ps1 -Output reports/baseline-$(Get-Date -Format yyyy-MM-dd).csv"
