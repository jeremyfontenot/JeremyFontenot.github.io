Identity & Access

Active Directory Migration Plan

Moved a flat lab domain into an OU, group, and GPO model. The new model is easier to manage and audit.

Windows Server

Problem

Users, workstations, and groups were mixed together. Delegation boundaries and policy targeting were unclear.

Solution

Designed a staged OU migration. Standardized security group naming. Documented GPO validation steps.

Outcome

Produced a supportable identity structure. Added rollback notes, a provisioning script, and a change record.

Technical details, artifacts, and lessons learned
Root Domain: lab.local
OU=Corp
  OU=Users
  OU=Computers
  OU=Groups
OU=Service
  OU=ServiceAccounts
  OU=Admin

Evidence

  • ad-migration-plan.md (example artifact)
  • ad-migration-plan-2023-06-21.yaml (example metadata)
  • Provisioning script
  • gpo-template-StandardDesktop.xml (example artifact)

Lessons learned

OU design should follow operational support boundaries, not just department names. Validation and rollback notes matter as much as the target structure.

Cloud Administration

Microsoft 365 Personal Cloud Lab

Built a Microsoft 365 lab across Entra ID, Exchange Online, SharePoint, Teams, Intune, and baseline reporting.

Microsoft 365

Problem

Entry-level administrators need safe M365 practice. They also need to avoid production tenant risk.

Solution

Documented tenant layout and baseline controls. Defined service ownership and repeatable evidence collection.

Outcome

Troubleshooting

Incident Root Cause Analysis

Investigated authentication failures. Isolated DNS misconfiguration. Documented remediation and validated recovery.

RCA

Problem

Users had intermittent sign-in failures. Domain service resolution was inconsistent.

Solution

Reviewed logs and tested DNS resolution. Corrected forwarder settings. Recorded change evidence.

Outcome

Created a clear RCA with timeline, impact, root cause, remediation, and prevention actions.

Technical details, artifacts, and lessons learned
2025-02-10 08:42 users report intermittent auth failures
2025-02-10 09:05 DNS query failures reproduced
2025-02-10 09:22 forwarder mismatch identified
2025-02-10 09:41 remediation applied
2025-02-10 10:10 validation complete

Evidence

  • rca-2025-02-10.md (example artifact)
  • incident-2025-02-10-logs.txt (example log)
  • rca-run-2025-02-10.log (example artifact)
  • remediation-disable-transportrule.ps1 (example script)

Lessons learned

The best troubleshooting evidence shows the path from symptom to test to fix. Documented timelines reduce ambiguity during handoff.

Automation

PowerShell User Provisioning Automation

Built a repeatable user onboarding workflow with input validation, template-based group assignment, and logging.

PowerShell

Problem

Manual onboarding is repetitive. It often causes inconsistent naming, group assignment, and license steps.

Solution

Created a readable PowerShell workflow. Used approved verbs, parameter validation, and auditable output.

Outcome

Produced reusable automation evidence aligned to service desk and junior administrator responsibilities.

Technical details, artifacts, and lessons learned
Import-Csv .\\new-hires.csv | ForEach-Object {
  New-ADUser -Name $_.DisplayName -SamAccountName $_.SamAccountName
  Add-ADGroupMember -Identity $_.RoleGroup -Members $_.SamAccountName
  Write-Information "Provisioned $($_.SamAccountName)"
}

Evidence

  • Provisioning script
  • automation-commit-abc123.md (example metadata)
  • automation-commit-abc123.patch (example artifact)
  • automation-run-2024-10-05.log (example log)

Lessons learned

Automation for entry-level administration should prioritize validation, readable logs, and safe defaults over clever shortcuts.
