Identity & Access

Active Directory Migration Plan

Modernized a flat lab domain structure into an OU, group, and GPO model that is easier to administer and audit.

Windows Server

Problem

Users, workstations, and groups were mixed together without clear delegation boundaries or policy targeting.

Solution

Designed staged OU migration, standardized security group naming, and documented GPO validation steps.

Outcome

Produced a supportable identity structure with rollback notes, provisioning script, and change record.

Technical details, artifacts, and lessons learned
Root Domain: lab.local
OU=Corp
  OU=Users
  OU=Computers
  OU=Groups
OU=Service
  OU=ServiceAccounts
  OU=Admin

Evidence

  • Migration plan
  • Change metadata
  • Provisioning script
  • GPO XML sample

Lessons learned

OU design should follow operational support boundaries, not just department names. Validation and rollback notes matter as much as the target structure.

Troubleshooting

Incident Root Cause Analysis

Investigated authentication failures, isolated DNS misconfiguration, documented remediation, and validated recovery.

RCA

Problem

Users experienced intermittent sign-in failures and inconsistent domain service resolution.

Solution

Reviewed logs, tested DNS resolution, corrected forwarder configuration, and recorded change evidence.

Outcome

Created a clear RCA with timeline, impact, root cause, remediation, and prevention actions.

Technical details, artifacts, and lessons learned
2025-02-10 08:42 users report intermittent auth failures
2025-02-10 09:05 DNS query failures reproduced
2025-02-10 09:22 forwarder mismatch identified
2025-02-10 09:41 remediation applied
2025-02-10 10:10 validation complete

Evidence

  • RCA report
  • Incident logs
  • Run log
  • Remediation script sample

Lessons learned

The best troubleshooting evidence shows the path from symptom to test to fix. Documented timelines reduce ambiguity during handoff.

PowerShell User Provisioning Automation

Built a repeatable user onboarding workflow with input validation, template-driven group assignment, and logging.

PowerShell

Problem

Manual onboarding is repetitive and prone to inconsistent naming, group assignment, and license steps.

Solution

Created a readable PowerShell workflow with approved verbs, parameter validation, and auditable output.

Outcome

Produced reusable automation evidence aligned to service desk and junior administrator responsibilities.

Technical details, artifacts, and lessons learned
Import-Csv .\\new-hires.csv | ForEach-Object {
  New-ADUser -Name $_.DisplayName -SamAccountName $_.SamAccountName
  Add-ADGroupMember -Identity $_.RoleGroup -Members $_.SamAccountName
  Write-Information "Provisioned $($_.SamAccountName)"
}

Evidence

  • Provisioning script
  • Automation change summary
  • Patch evidence
  • Run log

Lessons learned

Automation for entry-level administration should prioritize validation, readable logs, and safe defaults over clever shortcuts.
