Homelab architecture

This homelab mirrors a small-business environment: a Windows Server domain, file services, basic monitoring, and cloud-connected services. It’s designed to practice real workflows: user provisioning, access control, troubleshooting, and documentation.

Windows Server · AD DS File services Monitoring Cloud concepts PowerShell automation

High-level diagram

The diagram below shows the core components: clients, a domain controller, file server, and cloud services. It’s not meant to be exhaustive, but to communicate how traffic and responsibilities are split.

Client segment

Workstations joined to the domain, using DHCP/DNS from the server and mapped drives for file access.

Core services

Domain controller providing AD DS, DNS, and DHCP. File server hosting departmental shares and home folders.

Cloud & monitoring

Microsoft 365 tenant, basic monitoring tools, and logging to validate services and track changes.

Identity & access

Users and groups in AD, mapped to shares and roles. Password policy and lockout thresholds configured.

Automation

PowerShell scripts for user creation, group membership, and basic reporting on AD and file usage.

Key components

Domain controller

AD DS, DNS, and DHCP for the lab network.

Includes OU structure, groups, and baseline GPOs for password policy and workstation settings.

File server

Departmental shares and home folders.

NTFS and share permissions mapped to groups, with an access matrix documented.

Cloud tenant

Microsoft 365 tenant for mail and collaboration.

Used to practice license assignment, mailbox basics, and identity concepts.

Practice scenarios

The homelab is used to rehearse real tasks: onboarding, access changes, troubleshooting, and small incidents. Each scenario is documented so it can be repeated and improved.

Onboarding

Create a new user, assign groups, and validate access.

Includes: PowerShell commands, screenshots of group membership, and mapped drive verification.

Access change

Adjust a user’s access to a departmental share.

Includes: updated access matrix, before/after screenshots, and notes on communication.

Troubleshooting

Resolve a DNS or login issue.

Includes: event log snippets, ping/nslookup outputs, and root cause notes.

Documentation

Update diagrams and notes after changes.

Ensures the lab stays aligned with what’s actually deployed.
