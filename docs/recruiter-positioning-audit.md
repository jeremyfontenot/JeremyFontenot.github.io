# Recruiter Positioning Audit

## Scope

Reviewed and updated the public recruiter-facing messaging on:

- `index.html`
- `projects.html`
- `proof.html`
- `resume.html`
- `contact.html`

The pass was content-only. Branding, colors, navigation, CSS layout architecture, project evidence links, and proof document paths were preserved.

## Homepage Findings

The homepage previously led with repository governance, validation systems, proof handling, CI/CD language, and evidence operations before clearly explaining why Jeremy is a fit for IT support and systems administration roles.

Primary risk: recruiters and hiring managers could understand the site as a documentation platform before understanding Jeremy as an experienced Service Desk Technician, Enterprise IT Support Professional, Microsoft 365 administrator candidate, and systems administration candidate.

## Homepage Changes

- Rewrote the hero section around immediate role fit: service desk, enterprise IT support, Microsoft 365, Entra ID, Active Directory, endpoint support, PowerShell, and infrastructure operations.
- Added the requested `At A Glance` positioning using the existing section structure, including 4+ years of IT experience based on resume date ranges.
- Reframed `Technical Focus` as `Core Technical Strengths`.
- Replaced governance-heavy homepage messaging with `Why Teams Hire Me`.
- Repositioned featured work around support-to-administrator capability instead of source access alone.
- Updated footer status language away from CI/CD and repository health toward recruiter-visible IT capabilities.

## Resume Page Changes

- Rewrote third-person phrasing into first-person professional language.
- Clarified role focus around enterprise service desk, IT support, Microsoft operations, and systems administration progression.
- Preserved truthful positioning and avoided senior-level overstatement.

## Projects Page Changes

- Reworked the introduction so projects map directly to Service Desk, Systems Administration, Infrastructure, Microsoft 365, and Automation.
- Preserved existing project evidence and project page links.
- Improved reviewer framing for hiring managers and technical screeners.

## Proof Page Changes

- Reduced governance-heavy wording.
- Explained proof as a recruiter-friendly way to verify resume and project claims.
- Preserved all proof and evidence document links.

## Contact Page Changes

- Improved the call to action for recruiters, hiring managers, and IT leaders.
- Clarified target opportunities: enterprise service desk, IT support, Microsoft 365 operations, junior systems administration, and infrastructure operations support.

## Positioning Guardrails

- No fabricated certifications, metrics, production claims, or senior-level role claims were added.
- Microsoft 365 language remains framed around administration exposure and preserved lab evidence.
- Systems administration language remains progression-oriented.
- Evidence links remain available for verification without overwhelming first-time recruiters.
