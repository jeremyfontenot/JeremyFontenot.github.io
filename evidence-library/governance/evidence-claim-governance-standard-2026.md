# Evidence Claim Governance Standard — 2026

## Purpose
Define how portfolio claims are connected to repository evidence.

## Standard
1. Claims must link to a repository-local document, diagram, script, validation record, screenshot, or preserved artifact.
2. Historical environments must be identified as historical, preserved, validation, or demonstration environments.
3. Dismantled or canceled services must never be represented as live systems.
4. If evidence is weak, the public page should use conservative wording and point to a documentation summary rather than exaggerating.
5. Public cards should prioritize reviewer verification over marketing language.

## Applied controls
- Microsoft 365 tenant references are labeled historical.
- Home lab references are labeled dismantled / historical.
- Proof links are repository-relative.
- Recruiter journey follows Home → Dashboard → Projects → Proof → Resume → Contact.
